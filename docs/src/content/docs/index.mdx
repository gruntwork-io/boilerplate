## How it works

Define a `boilerplate.yml` with your variables, write Go-template files, and run one command.
Boilerplate handles prompts, defaults, validation, and rendering.

<Tabs>
  <TabItem label="1. Define variables">
```yaml
# boilerplate.yml
variables:
  - name: ProjectName
    description: The name of your project
    type: string

  - name: EnableDocker
    description: Include Docker configuration?
    type: bool
    default: true
```
  </TabItem>
  <TabItem label="2. Write templates">
```text title="README.md"
# {{.ProjectName}}

Welcome to {{.ProjectName}}!

{{if .EnableDocker}}
## Docker

Run with: `docker compose up`
{{end}}
```
  </TabItem>
  <TabItem label="3. Generate">
```bash
boilerplate \
  --template-url ./my-template \
  --output-folder ./my-project \
  --var ProjectName="My App"
```
  </TabItem>
  <TabItem label="4. Output">
```markdown title="my-project/README.md"
# My App

Welcome to My App!

## Docker

Run with: `docker compose up`
```
  </TabItem>
</Tabs>

## Why Boilerplate?

Boilerplate is designed for platform teams who need to generate infrastructure code, create app starter templates, enforce standards, and automate workflows across an entire organization.

<CardGrid>
  <Card title="Run it anywhere" icon="rocket">
    Boilerplate is a single Go binary with zero runtime dependencies. Download it, run it on your laptop, in CI/CD, or in a container. No Python, no Node, no version conflicts.
  </Card>
  <Card title="Easily capture variables" icon="setting">
    Boilerplate accepts variable values interactively, from the CLI, via YAML files, or through environment variables.
  </Card>
  <Card title="Build powerful templates" icon="puzzle">
    Boilerplate extends Go templates to include serious templating power including 40+ built-in functions, partials, conditionals, capturing shell command output, and more.
  </Card>
</CardGrid>

### And there's more

- **[Composable templates](/configuration/dependencies/)** — Build complex scaffolds from smaller, reusable modules using the dependency system.
- **[Variable validation](/configuration/variables/)** — Seven variable types with built-in validators including required, regex, email, URL, and semver.
- **[Hooks](/configuration/hooks/)** — Run shell commands before or after rendering to format code, install dependencies, or run tests.
- **[Remote templates](/advanced/remote-templates/)** — Fetch templates from Git repos, S3, GCS, or HTTP URLs.
- **[Snippets](/advanced/snippets/)** — Embed code from source files in generated docs to keep examples always in sync.
- **[Non-interactive mode](/cli/flags/)** — Run fully headless in CI/CD pipelines with `--non-interactive`, `--var-file`, and environment variables.
