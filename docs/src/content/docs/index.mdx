## What is Boilerplate?

Boilerplate is a tool for generating files and folders (boilerplate code) from a set of templates. It's
built by [Gruntwork](https://gruntwork.io) and is designed for:

- **Scaffolding** new projects, modules, and repositories from templates
- **Generating** configuration files with dynamic variable substitution
- **Embedding** code snippets from source files into documentation
- **Automating** repetitive file creation in CI/CD pipelines

## Key Features

<CardGrid stagger>
  <Card title="Flexible Variables" icon="pencil">
    Seven built-in types (string, int, float, bool, list, map, enum) with validations, defaults, and interactive prompts.
  </Card>
  <Card title="Go Template Engine" icon="setting">
    Full Go template syntax with 50+ helper functions from the Sprig library, plus custom helpers for snippets, shell commands, and more.
  </Card>
  <Card title="Template Dependencies" icon="puzzle">
    Chain multiple templates together with recursive dependencies, variable inheritance, conditional skipping, and loop-based rendering.
  </Card>
  <Card title="Hooks" icon="rocket">
    Run shell commands before and after template rendering, with full variable interpolation and conditional execution.
  </Card>
</CardGrid>

## Quick Example

A typical Boilerplate template has a `boilerplate.yml` config file and one or more template files:

```yaml
# boilerplate.yml
variables:
  - name: ProjectName
    description: The name of your project
    type: string

  - name: EnableDocker
    description: Include Docker configuration?
    type: bool
    default: true
```

```text
# {{.ProjectName}}

Welcome to {{.ProjectName}}!

{{if .EnableDocker}}
## Docker

Run with: `docker compose up`
{{end}}
```

Then run:

```bash
boilerplate \
  --template-url ./my-template \
  --output-folder ./my-project \
  --var ProjectName="My App"
```
